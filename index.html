Sylectra - An Intensely Basic CSS Dom Selector

(Seriously, you can't get any more simple)

I know—the pilgrimage to building a full-featured selector engine is a long and hard journey that few dare to attempt. However, a bit of free time gave me the opportunity to climb to a small hill and feel the breezes from the distant mountain where Sizzle, NWMatcher, and the rest of that elite breed dwell.

Sylectra is my feeble attempt at creating a selector engine. You should probably never use it, but you might want to read through the code. Actually, that might be a bad idea; see, I've never read through the source code of a real selector engine, so I have so idea how the well-known ones actually operate.

Sylectra works with only the simplest of selector:

  • Ids : #content
  • Classes : .tabbed-container
  • Elements : div
  • Element + (id || class) : div#sidebar or li.product
  • Any combination of the above : #main .panels li a

Yes, that's all; I said it was basic.

To further discourage you from using Sylectra, notice that it does not support direct children (>) selectors, attribute selectors, multiple (comma-delimited) selectors (though I may add that). Of course, it doesn't work in browsers that don't support document.getElementsByClassName.

To use Sylectra (and remember: don't), you just call the SYLECTRA function, passing in a CSS selector. You'll get back an element or an array of elements (or a NodeList, but it's an array-like object, so for all intents and purposes, use it like an array). It's that easy.
